# 1. Purpose

The database is responsible for storing all persistent information generated by Astravon Live Arena.

The AI Engine performs real-time analysis but does **not** permanently store data. Instead, it sends processed results to the backend, which validates and stores them in PostgreSQL.

The database enables:

- Event management
- Historical crowd analytics
- Alert logging
- Risk monitoring
- Report generation
- Future predictive analytics

The database stores

- Crowd statistics

# 16. Query Examples

Current Event

```
SELECT *
FROM events;
```

Latest Statistics

```
SELECT *
FROM statistics
ORDER BY timestamp DESC
LIMIT 1;
```

Current Alerts

```
SELECT *
FROM alerts
WHERE resolved = FALSE;
```

Highest Crowd Count

```
SELECT MAX(people_count)
FROM statistics;
```

Highest Temperature

```
SELECT MAX(temperature)
FROM statistics;
```

Average Risk

```
SELECT AVG(risk_score)
FROM statistics;
```

# 19. Database Security

Recommended practices

- Use strong passwords
- Store credentials in `.env`
- Never commit passwords
- Restrict database users
- Enable SSL for production
- Validate all input
- Prevent SQL Injection using SQLAlchemy ORM

# 26. Database Design Principles

The database follows these principles:

- Normalized where practical (3NF)
- Minimal data redundancy
- Clear relationships
- Fast read operations
- Easy future expansion
- Separation of AI processing from persistent storage
- Maintainable schema design

---

# 27. Summary

The PostgreSQL database serves as the persistent storage layer of Astravon Live Arena.

It provides a reliable foundation for recording event information, crowd statistics, alerts, and reports while remaining lightweight enough for an academic prototype. By separating transient AI processing from long-term storage, the architecture stays modular, scalable, and ready for future enhancements such as multi-camera deployments, IoT integration, predictive analytics, and cloud-based event management.
